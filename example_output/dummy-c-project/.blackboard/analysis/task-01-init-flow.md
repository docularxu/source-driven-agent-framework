# Task 01: init-flow — main() 初始化调用链追踪

## 分析目标
追踪 main() 中的初始化流程，查清实际操作的硬件寄存器地址。

## 关注点（严格限定）
1. main() 调用了什么？通过什么机制？
2. 实际执行的初始化函数是哪个？在哪个文件？
3. 该函数操作的寄存器地址是什么？如何计算得出？

## 代码范围
- main.c（入口）
- driver_api.h（契约定义）
- usb_driver.c（驱动实现）

## 注意事项
- 注意函数指针间接调用
- 注意宏定义的地址计算
- 忽略 legacy_init_do_not_use（废弃接口）

## 输出要求
- 完整调用链（含文件名和行号）
- 寄存器地址的计算过程
- 标注所有关键证据（源码引用）
